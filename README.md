# Графік будівельних робіт — Gantt-борд

Легкий веб-застосунок для планування робіт 4 паралельних будівельних бригад.
Перетягуйте картки проєктів, тягніть за правий край картки, щоб змінити
тривалість — усі наступні проєкти в тій самій бригаді автоматично зсуваються
без "дірок" у графіку.

## Стек
Next.js 14 (App Router) + React + Tailwind CSS + dnd-kit (drag&drop) +
Supabase (спільна база даних і realtime-синхронізація).

## Локальний запуск

```bash
npm install
npm run dev
```

Відкрийте http://localhost:3000 — застосунок одразу працює з демо-даними
навіть без Supabase (локальний режим, без збереження між сесіями).

## Підключення Supabase (щоб колеги бачили ту саму версію)

1. Створіть проєкт на https://supabase.com (безкоштовний тариф достатній).
2. У розділі **SQL Editor** виконайте вміст файлу `supabase/schema.sql` —
   він створить таблиці `crews` і `projects`, налаштує публічний доступ на
   читання/запис та увімкне realtime.
3. У розділі **Project Settings → API** скопіюйте `Project URL` та
   `anon public` ключ.
4. Скопіюйте `.env.example` у `.env.local` і вставте туди ці значення:

   ```
   NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-public-key
   ```

5. Перезапустіть `npm run dev` — при першому відкритті застосунок сам
   заповнить базу демо-даними. Кнопка «Скинути до демо-даних» очищає базу
   і завантажує демо-набір знову.

> Політики доступу в `schema.sql` дозволяють читання й запис усім, хто має
> посилання на застосунок (без логіну) — це навмисно спрощено для швидкого
> спільного доступу командою. Якщо потрібна авторизація користувачів,
> додайте Supabase Auth і звузьте RLS-політики.

## Деплой на Vercel

1. Залийте проєкт у GitHub-репозиторій.
2. На https://vercel.com натисніть **New Project** → виберіть репозиторій.
3. У розділі **Environment Variables** додайте `NEXT_PUBLIC_SUPABASE_URL`
   та `NEXT_PUBLIC_SUPABASE_ANON_KEY` (ті самі значення, що й у `.env.local`).
4. Натисніть **Deploy** — за замовчуванням Vercel визначить, що це Next.js
   проєкт, і збере його автоматично.
5. Готово — дайте колегам посилання на застосунок, усі бачитимуть і
   редагуватимуть один і той самий графік у реальному часі.

## Основний функціонал

- 4 паралельні доріжки-бригади з послідовними проєктами без "дірок".
- Перетягування карток для зміни порядку в бригаді або переносу між бригадами.
- Зміна тривалості перетягуванням правого краю картки — наступні проєкти
  каскадно зсуваються.
- Додавання / редагування (назва, тривалість, колір, бригада) / видалення
  проєктів через модальне вікно.
- Кнопка «Скинути до демо-даних» для швидкого тестування.
- Індикатор «Синхронізовано / Локально» у шапці показує, чи підключено
  Supabase.

## Структура проєкту

```
app/                 сторінки Next.js (App Router)
components/          GanttBoard, LaneRow, ProjectCard, ProjectDialog, TimelineRuler
lib/                 типи, каскадна логіка розкладу, Supabase-клієнт, стан дошки
supabase/schema.sql  SQL-схема таблиць crews і projects
```
